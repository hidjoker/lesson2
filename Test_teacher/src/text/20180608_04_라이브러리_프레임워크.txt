
 + 라이브러리, Library
  - 프로그래밍에서 활용가능한 도구들의 모음, 집합
   ( 메소드, 함수, 클래스 등등 )

  - 특정 기능에 대한 API(모음집)



 + 프레임워크, Framework
  - 뼈대, 기반구조

  - 문제를 해결하기 위해 상호 협력하는 클래스와 인터페이스의 집합



 + 라이브러리 vs 프레임워크

  - 라이브러리와 달리 프레임워크는 API 사용절차까지 포함한다

  - 라이브러리는 응용프로그램이 코드의 흐름을 직접 제어한다
  - 프레임워크는 코드의 흐름을 프레임워크가 제어한다


  - 직접 호출해서 사용하는 시스템이면 라이브러리
  - 코드를 만들어두면 시스템이 가져다가 사용하면 프레임워크
	=> IoC, Inversion Of Control, 제어의 역전
